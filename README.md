# MobX State Management Practice Project

Цей проєкт містить серію практичних задач для глибокого вивчення **MobX 6** та його інтеграції з **React**. Основна увага приділена роботі з реактивними масивами, механізмам автоматичного очищення стану (auto-cleanup) та оптимізації рендеру.

## 🛠 Технологічний стек

- **React** (Vite + TS)
- **MobX 6** (Decorators, Observable, Actions, Computed)
- **MobX React Lite** (`observer`)
- **TypeScript** (Strong Typing)

---

## 📋 Реалізовані модулі

### 1. Typing Speed Store (`TypingSpeedStore`)

Система вимірювання швидкості друку в реальному часі (WPM).

- **Функціонал:** Реєстрація натискань клавіш, розрахунок середньої швидкості за ковзним вікном у 5 секунд.
- **Idle Detection:** Автоматичне перемикання статусу в `Idle`, якщо користувач не друкує понад 2 секунди.
- **Ключові концепти:** Розрахунок формул у `@computed`, глобальні `EventListener` у React, поєднання `setInterval` та `setTimeout` для керування часом.

### 2. Emoji Reaction Store (`EmojiStore`)

Система відстеження активності емодзі з логікою анти-спаму.

- **Функціонал:** Додавання емодзі з автоматичним видаленням запису через 3 секунди.
- **Spam Detection:** Блокування інтерфейсу, якщо інтенсивність натискань занадто висока.
- **Ключові концепти:** `@observable` масиви, асинхронні зміни через `runInAction`.

### 3. Network Diagnostics Store (`InternetSpeedStore`)

Моніторинг стабільності з'єднання (Ping).

- **Функціонал:** Збереження історії пінгу та динамічна класифікація якості зв'язку (`Excellent` / `Unstable` / `Bad`).
- **Ключові концепти:** Обробка масивів через `.reduce()`, робота з математичними округленнями в реактивному середовищі.

### 4. Smart Device Queue Store (`SmartLightStore`)

Імітація черги команд для IoT-пристрою.

- **Функціонал:** Обмежена черга (max 3 команди) з послідовною обробкою та імітацією затримки "заліза".
- **Ключові концепти:** FIFO логіка, використання `observable.ref` для оптимізації черги.

## 🏗 Архітектурні принципи

У проєкті реалізовано наступні підходи MobX:

1. **Reactive Tracking:** Використання декораторів (`@observable`, `@action`, `@computed`) разом із `makeObservable(this)` для чіткої структури класу.
2. **Auto-Cleanup Pattern:** Реалізація механізмів самоочищення масивів через `setInterval` або `setTimeout`, що дозволяє уникнути витоків пам'яті та тримати стан актуальним.
3. **Async Integrity:** Будь-яка зміна стану після завершення таймерів або асинхронних операцій виконується виключно всередині `runInAction`.
4. **Computed Efficiency:** Важкі розрахунки (як-от WPM або фільтрація історії) винесені в `@computed` геттери для кешування результатів.
